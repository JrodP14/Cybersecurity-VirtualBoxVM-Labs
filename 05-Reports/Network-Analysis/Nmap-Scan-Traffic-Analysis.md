# Nmap Scan Traffic Analysis

## Objective

Analyze network reconnaissance traffic generated by Nmap scans against the Metasploitable2 virtual machine and identify exposed services, open ports, and potential security risks.

---

## Environment

| System | Role |
|---|---|
| Kali Linux | Scanning Host |
| Metasploitable2 | Target Host |
| VirtualBox | Virtualization Platform |
| Nmap | Network Scanning Tool |

---

## Tools Used

- Nmap 7.99
- Kali Linux
- Metasploitable2
- VirtualBox

---

## Target Information

| Target | IP Address |
|---|---|
| Metasploitable2 | 10.0.2.4 |

---

# Host Discovery Scan

## Command Used

```bash
sudo nmap -sn 10.0.2.0/24
```

## Purpose

The host discovery scan was used to identify live systems on the local network without performing a full port scan.

## Findings

The scan identified multiple active systems within the subnet:

| IP Address | Description |
|---|---|
| 10.0.2.1 | Gateway |
| 10.0.2.3 | Kali Linux |
| 10.0.2.4 | Metasploitable2 |

## Analysis

- ICMP and ARP responses confirmed active hosts
- Oracle VirtualBox MAC addresses were detected
- Network enumeration successfully identified reachable systems

---

# TCP Port Scan

## Command Used

```bash
sudo nmap -Pn 10.0.2.4
```

## Purpose

Perform a TCP port scan against the Metasploitable2 target to identify open services and listening ports.

## Findings

The scan identified numerous exposed services:

| Port | Service |
|---|---|
| 21 | FTP |
| 22 | SSH |
| 23 | Telnet |
| 25 | SMTP |
| 53 | DNS |
| 80 | HTTP |
| 139 | NetBIOS |
| 445 | SMB |
| 3306 | MySQL |
| 5432 | PostgreSQL |
| 5900 | VNC |
| 8180 | Apache Tomcat |

## Analysis

- Multiple legacy and insecure services were exposed
- Telnet and FTP services transmitted data insecurely
- SMB services increased attack surface exposure
- Database services were publicly reachable
- Several services are known to contain historical vulnerabilities in Metasploitable2

## Security Impact

Attackers could use exposed services for:

- Enumeration
- Credential attacks
- Remote exploitation
- Service fingerprinting
- Vulnerability discovery

## Analysis

- Several outdated services were identified
- Legacy software versions are associated with known CVEs
- Service fingerprinting provides attackers with valuable reconnaissance data

---

# Aggressive Scan Analysis

## Command Used

```bash
sudo nmap -A 10.0.2.4
```

## Purpose

Perform advanced enumeration including:

- OS detection
- Service detection
- Script scanning
- Version detection
- Traceroute analysis

## Findings

The aggressive scan identified:

- Linux operating system fingerprints
- Apache HTTP server details
- Samba workgroup information
- RPC services
- DNS server details
- VNC services
- Apache Tomcat services

The scan also identified:

- Metasploitable-specific vulnerable services
- UnrealIRCd
- Java RMI registry
- Bind shell service on port 1524

## Analysis

Aggressive scanning provides deep reconnaissance information that attackers can use for:

- Vulnerability targeting
- Exploit selection
- Service enumeration
- Network mapping

## Security Impact

Detailed scan information increases the likelihood of successful exploitation by exposing:

- Operating system details
- Service versions
- Vulnerable applications
- Legacy protocols

# Targeted Service Analysis

## Command Used

```bash
sudo nmap -p 21,22,80 10.0.2.4
```

## Purpose

Perform targeted analysis against specific high-value services.

## Findings

| Port | Service | Status |
|---|---|---|
| 21 | FTP | Open |
| 22 | SSH | Open |
| 80 | HTTP | Open |

## Analysis

Targeted scanning reduces network noise while focusing on important services commonly used during reconnaissance and exploitation phases.

### FTP (21)

- File transfer service exposed
- Legacy FTP protocol transmits data unencrypted

### SSH (22)

- Secure remote administration service
- Older OpenSSH version detected

### HTTP (80)

- Web server exposed
- Apache web application services identified

# Security Findings

## High-Risk Services Identified

| Service | Risk |
|---|---|
| Telnet | Unencrypted communication |
| FTP | Plaintext authentication |
| SMB | Vulnerable file sharing exposure |
| VNC | Remote desktop exposure |

# Mitigation Recommendations

- Disable unnecessary services
- Restrict access using firewalls
- Remove legacy protocols such as Telnet and FTP
- Upgrade outdated software versions
- Segment vulnerable systems from production networks
- Implement intrusion detection monitoring
- Limit external exposure of database services

---

# Key Takeaways

- Nmap provides powerful reconnaissance capabilities
- Service enumeration reveals attack surface exposure
- Legacy services significantly increase security risk
- Banner grabbing exposes valuable system information
- Aggressive scanning identifies vulnerable applications
- Network reconnaissance is a critical phase of penetration testing and offensive security operations
