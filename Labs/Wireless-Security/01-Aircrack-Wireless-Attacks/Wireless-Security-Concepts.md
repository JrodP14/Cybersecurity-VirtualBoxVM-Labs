# Wireless Security Concepts

## Overview

This document covers defensive wireless-security concepts, secure Wi-Fi practices, and authentication-security considerations reviewed during the wireless-security lab.The focus of this lab was defensive awareness and secure wireless-network configuration.

---

# Common Wireless Security Risks

Potential risks include:
- weak passwords
- rogue access points
- unauthorized access
- outdated encryption
- insecure configurations

---

# Common Wireless Encryption Types

| Protocol | Security Level |
|---|---|
| WEP | Insecure |
| WPA | Legacy |
| WPA2 | Common secure standard |
| WPA3 | Improved modern security |

---

# Defensive Security Recommendations

Recommended practices:
- use WPA2/WPA3
- disable WPS
- use strong passphrases
- update firmware
- monitor connected devices

---

# Strong Passphrase Concepts

Strong wireless passwords should:
- be lengthy
- avoid predictable patterns
- include randomness
- avoid reuse

---

# Wireless Monitoring

Security teams should monitor:
- unknown devices
- rogue access points
- unusual traffic
- failed authentication attempts
